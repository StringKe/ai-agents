# /verification-before-completion — 完成前验证

## 铁律

**没有验证证据就不宣称完成。**

在声称任何任务"已完成"之前，必须运行验证命令并展示输出。

## 验证清单

在宣称完成之前，按顺序执行：

1. **编译通过**：运行项目的编译命令，输出无错误
2. **测试通过**：运行项目的测试命令，所有测试绿色
3. **功能验证**：如果改动影响了可观察的行为，展示行为正确的证据
4. **无回归**：相关的现有测试仍然通过

## 证据标准

可接受的证据：
- 命令行输出（编译、测试、lint 的完整输出）
- 程序运行截图或输出
- 对比修改前后的行为差异

不可接受的证据：
- "我检查了代码，看起来没问题"
- "根据我的理解，这应该能工作"
- "测试应该会通过"（没有实际运行）

## 执行方式

```bash
# 示例验证序列
npm run lint && npm run build && npm test
./gradlew check
cargo clippy && cargo test
go vet ./... && go test ./...
```

如果任何验证步骤失败，修复后重新验证，直到全部通过。
