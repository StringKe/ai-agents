# /dispatching-parallel-agents — 并行代理调度

当面对多个相互独立的任务时，为每个任务派遣一个独立的子代理并行执行。

## 适用场景

- 多个独立的 bug 需要同时修复
- 多个不相关的文件需要同时修改
- 需要同时从多个方向调研一个问题
- plan.md 中有多个互不依赖的任务

## 不适用场景

- 任务之间有先后依赖
- 多个任务修改同一个文件
- 需要统一的设计决策

## 流程

### 第一步：识别独立问题域

分析所有待处理的任务，将它们划分为独立的问题域。独立性判断标准：

- 修改不同的文件
- 不共享可变状态
- 不需要彼此的输出作为输入

### 第二步：为每个问题域准备上下文

为每个子代理准备独立的上下文包：

- 任务描述
- 相关文件列表
- 预期产出
- 约束条件

### 第三步：并行派遣

使用 Agent 工具同时派遣多个子代理。每个子代理的 prompt 包含：

1. 任务描述和预期产出
2. 相关文件路径
3. 约束条件（不修改其他文件、遵循项目约定）
4. 完成标准

关键：在单个回复中发起所有 Agent 调用，确保真正的并行执行。

### 第四步：收集和整合结果

所有子代理完成后：

1. 收集每个子代理的执行结果
2. 检查是否有冲突（不应该有，但需要确认）
3. 运行编译和测试确认所有改动兼容
4. 向用户汇总报告

## 代理数量指南

- 2 到 4 个并行代理：适合大多数场景
- 5+ 个并行代理：仅在任务真正独立且简单时使用
- 如果需要更多，考虑分批执行
