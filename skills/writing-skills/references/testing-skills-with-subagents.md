# 使用子代理测试技能

## 测试原理

派遣一个子代理来模拟"使用技能的 AI"，验证技能指令是否产生预期行为。

## 测试流程

### 1. 准备测试场景

```
场景 ID：test-001
触发输入：用户说"帮我调试这个 API 返回 500 的问题"
预期匹配技能：systematic-debugging
预期首步行为：收集错误信息和日志
验证方法：检查子代理是否先调查再修复
```

### 2. 派遣测试子代理

使用 Agent 工具，给子代理以下 prompt：

```
你将扮演一个 AI 助手。以下是你可用的技能指令：

[粘贴 SKILL.md 内容]

用户输入："[测试场景的触发输入]"

请按照技能指令执行。不要实际修改任何文件，只描述你会做什么。
```

### 3. 验证结果

检查子代理的回复：
- 是否按照技能流程的步骤执行
- 是否遵守了铁律和约束
- 是否使用了正确的工具
- 流程是否在预期的点终止

### 4. 常见问题

- **子代理跳过步骤**：说明步骤间的转换条件不够明确，需要强化
- **子代理做了不该做的事**：说明禁止行为的描述不够醒目，考虑加粗或用独立的"不做的事"章节
- **子代理不确定何时结束**：说明完成标准不明确

## 回归测试

修改 SKILL.md 后，重新运行所有测试场景，确认没有引入新的问题。
