# 根因追踪方法

## 五个为什么

对每个问题连续问五次"为什么"，直到触及根本原因。

**示例**：
- 问题：API 返回 500
- 为什么？数据库查询抛出异常
- 为什么？SQL 语法错误
- 为什么？字段名拼写错误
- 为什么？重命名字段时遗漏了这个查询
- 为什么？没有测试覆盖这个查询路径

根因：缺少测试覆盖。修复：写测试 + 修正字段名。

## 差异法

对比正常情况和异常情况的差异，锁定变量。

1. 找到一个正常运行的案例
2. 找到一个失败的案例
3. 系统性地对比两者的差异（输入、环境、配置、版本）
4. 逐一排除差异，直到找到关键差异

## 二分法

当问题在某个时间点之后出现时，用 git bisect 缩小范围。

```bash
git bisect start
git bisect bad HEAD
git bisect good <known-good-commit>
# git 会自动 checkout 中间的 commit
# 测试是否有问题，然后标记 good/bad
git bisect good  # 或 git bisect bad
# 重复直到找到引入问题的 commit
git bisect reset
```

## 隔离法

将系统拆分为独立组件，逐个测试。

1. 列出问题可能涉及的所有组件
2. 用最简单的输入测试每个组件
3. 找到失败的组件
4. 在该组件内部继续隔离

## 时间线法

重建问题发生的完整时间线。

1. 收集所有可用的日志（应用日志、系统日志、网络日志）
2. 按时间排序
3. 标注异常事件
4. 寻找因果关系链
